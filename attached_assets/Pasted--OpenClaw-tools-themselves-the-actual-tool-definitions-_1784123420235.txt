"Direct Google Drive / Gmail / Docs / Sheets — not implemented as direct routes"

Wrong. artifacts/api-server/src/routes/integrations.ts has live direct routes backed by artifacts/api-server/src/lib/google.ts. Auth is a single Google OAuth credential (client_id + client_secret + refresh_token → auto-refreshed access token, or a pasted short-lived access_token). All four Google products share the same credential.

Route	What it does
GET /integrations/gmail/messages	Lists up to 50 messages. Supports Gmail query filter via ?q=. Returns id, snippet, subject, from, date per message.
GET /integrations/sheets/:id	Reads a range from a Google Sheet by spreadsheet ID. Default range A1:Z100, overridable via ?range=.
GET /integrations/docs/:id	Fetches a Google Doc by ID, extracts plain text. Returns {title, text}.
GET /integrations/drive/files	Lists up to 25 Drive files (id, name, mimeType, modifiedTime, webViewLink). Supports Drive query via ?q=.
GET /integrations/youtube/search	YouTube Data API search (up to 10 video results). Requires a separate youtube.api_key credential.
GET /integrations/instagram/media	Lists Instagram media (Meta Graph API). Requires instagram.access_token.
These are read-only routes. There are no write routes for Sheets, Docs, or Drive in this file. Publish to Instagram goes through Composio (/social/publish/:id), not this route.

"Scheduled post auto-publishing cron — no running cron worker"

Wrong. There are two independent implementations, both present in this repo:

artifacts/api-server/src/social-cron.ts — embedded directly inside the API server process. startSocialCron(port) is called once at server boot. Queries the database directly (no HTTP hop for the due-post lookup), fires POST /api/social/publish/:id internally every 60 seconds. Also handles recurring posts: after a successful publish, if interval_hours is set on the post, it reschedules scheduledAt = now + intervalHours.

scripts/social-media-worker.mjs — a standalone Node.js worker script meant to run separately. Self-signs a wt_session cookie from SESSION_SECRET so it can call the authenticated API. Polls GET /api/social/due every 60 seconds and publishes + reschedules via the same /api/social/publish/:id and /api/social/schedule/:id endpoints.

Whether the standalone worker is configured as a workflow in this repo: not verified — I did not check artifact.toml or .replit for a social-media-worker workflow entry.
